以下の観点でZenn記事をレビューしてください。

レビュー対象の記事ファイルを指定してもらう場合は聞いてください。

## チェック項目

### 1. 技術的正確性

- [ ] コードサンプルは動作するか
- [ ] 技術用語は正しく使われているか
- [ ] 非推奨の機能を使っていないか
- [ ] 参考リンクは有効か

### 2. 文章品質

- [ ] 誤字脱字はないか
- [ ] わかりやすい表現か（専門用語の説明はあるか）
- [ ] 段落構成は適切か
- [ ] 読者ターゲットに合っているか

### 3. Zenn記法

- [ ] メタデータ（title, emoji, type, topics）は正しいか
- [ ] 画像パスは正しいか（`/images/[slug]/`）
- [ ] コードブロックに言語指定があるか
- [ ] Zenn独自記法（:::message など）は正しく使われているか

### 4. Markdown品質

- [ ] Markdownlintルールに準拠しているか（`npm run lint:md` でチェック）
- [ ] 見出しレベルは適切か（H1は使わない）
- [ ] リンクは正しく設定されているか

## レビュー手順

1. 記事ファイルを読み込む
2. 上記チェック項目を確認
3. 問題点をリスト形式で報告
4. 修正提案を具体的に示す
5. 必要であれば `npm run lint:md` を実行してMarkdownエラーも確認
